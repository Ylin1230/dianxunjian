# 点巡检系统测试用例说明

编写日期：2026-05-11

## 交付内容

- `点巡检系统测试用例.xlsx`：测试人员执行主文件，包含总览、百数云入口检查清单、功能用例、API 与数据校验、非功能与回归、测试数据准备。
- 本说明文件：快速说明测试范围、优先级和建议执行顺序。

## 范围说明

本用例基于项目根目录中的自定义 HTML 页面和 `backend/src` 后端接口整理，但执行入口按实际部署方式优化为“从百数云左侧菜单点击外链页面”。后端当前为服务器常启状态，因此不把启动后端、验证 server.js 路由、手工访问 /api 作为普通测试人员的主用例。

用户截图中的蓝色文档类百数云底表不作为主测模块，只在需要核对数据落库、下拉选项或联动结果时抽查。

## 建议执行顺序

1. 先执行“百数云入口检查清单”：从百数云菜单逐个点击外链页面，确认页面能打开且标题/按钮/列表正常。
2. 执行设备/危化品台账 P0，准备测试对象。
3. 先执行点检规则 P0，配置日检/周检/月检等规则；再执行点检标准 P0，在标准中选择对应执行规则并生成点检任务。
4. 执行点检报工 P0，完成正常与异常闭环。
5. 执行月度点检报表 P1，核对设备可搜索单选、月份选择、点检明细、异常隐患和导出/打印。
6. 执行安全检查 P0，完成模板、任务、H5 现场照片/签名提交、隐患生成。
7. 执行相关方培训 P0，验证公开填报和管理端查询。
8. 补充 P1/P2、接口排查参考、非功能、兼容和回归用例。
