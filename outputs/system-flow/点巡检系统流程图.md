# 点巡检系统流程图

编写日期：2026-05-11

本文档用于客户演示和内部讲解。建议先讲“系统总览”，再按“点检闭环、安全检查闭环、相关方培训闭环、数据关系”展开。

## 1. 系统总览

客户视角可以理解为：百数云负责统一入口、底表和流程，自定义页面负责把现场业务做成更顺手的操作界面，后端服务负责和百数云 OpenAPI、二维码、附件、任务底表打通。

```mermaid
flowchart LR
  U["用户/测试人员/现场人员"] --> P["百数云门户<br/>左侧菜单入口"]

  P --> D["基础台账页面<br/>设备台账、危化品台账"]
  P --> S["点检配置页面<br/>点检标准、点检规则"]
  P --> R["点检报工页面<br/>扫码、填写、提交"]
  P --> C["安全检查页面<br/>模板、任务、移动端执行、记录"]
  P --> T["相关方培训页面<br/>记录管理、二维码下载、扫码填报端"]

  D --> API["后端服务<br/>接口代理、二维码、附件、任务同步"]
  S --> API
  R --> API
  C --> API
  T --> API

  API --> BES["百数云 OpenAPI<br/>主表、明细表、流程、附件"]
  API --> DB["点检任务底表<br/>百数云或 MySQL"]
  BES --> FLOW["百数云流程中心<br/>隐患整改、验收闭环"]
```

## 2. 点检业务闭环

这一张适合讲“系统为什么能自动生成任务、现场为什么扫码就能报工”。

```mermaid
flowchart TD
  A["基础数据准备"] --> A1["设备台账<br/>设备编号、类型、位置、责任人、二维码"]
  A --> A2["危化品台账<br/>危化品编号、存放位置、事故类型、二维码"]
  A --> A3["基础底表<br/>设备类型、场地、包装形式、事故类型、人员"]

  A1 --> B["点检标准配置"]
  A2 --> B
  A3 --> B

  B --> B1["标准主表<br/>标准名称、执行规则、版本、生效日期"]
  B --> B2["标准明细<br/>普通项、状态项、抄表项、异常必填要求"]
  B --> B3["适用范围<br/>关联设备或危化品"]

  B1 --> C["点检规则配置"]
  B2 --> C
  B3 --> C
  C --> C1["日检/周检/月检/自定义周期"]
  C1 --> D["自动生成点检任务<br/>任务日期、对象、标准、应报工项数"]

  D --> E["现场扫码报工"]
  E --> E1["扫码设备/危化品/地点二维码"]
  E1 --> E2["系统匹配今日待执行任务"]
  E2 --> E3["填写点检项<br/>正常、异常、抄表数值、照片"]

  E3 --> F{"是否存在异常"}
  F -- "否" --> G["报工完成<br/>任务状态=已完成"]
  F -- "是" --> H["生成隐患整改工单<br/>进入百数云流程中心"]
  H --> I["整改、验收、关闭"]
  I --> J["形成完整追溯<br/>任务、报工、明细、隐患"]
  G --> J
```

## 3. 点检报工现场流程

这一张适合给现场主管或一线人员演示。

```mermaid
flowchart TD
  A["进入点检报工页面"] --> B["选择或绑定执行人"]
  B --> C["扫码或输入码值"]
  C --> D{"是否识别到今日任务"}
  D -- "否" --> D1["提示今日暂无任务<br/>或选择其他任务"]
  D -- "是" --> E["加载点检明细"]

  E --> F["逐项填写结果"]
  F --> F1["普通项<br/>录入正常/异常"]
  F --> F2["状态项<br/>选择状态按钮"]
  F --> F3["抄表项<br/>录入数值并自动判定"]

  F1 --> G{"异常项是否已补充说明/照片"}
  F2 --> G
  F3 --> G

  G -- "缺少必填" --> G1["前端提示补充<br/>不允许提交"]
  G -- "满足要求" --> H["提交汇总"]
  H --> I["生成报工主表"]
  H --> J["生成报工明细表"]
  H --> K["更新点检任务状态"]
  H --> L{"是否异常"}
  L -- "异常" --> M["自动发起隐患整改"]
  L -- "正常" --> N["点检完成"]
  M --> N
```

## 4. 安全检查闭环

安全检查和点检类似，但起点是“检查模板”和“检查任务”，执行端偏移动化。

```mermaid
flowchart TD
  A["安全检查模板"] --> A1["模板主表<br/>检查类型、适用区域、默认部门/检查人"]
  A --> A2["检查项明细<br/>分类、检查项目、标准要求、结果类型、整改期限"]

  A1 --> B["创建检查任务"]
  A2 --> B
  B --> B1["任务编号、计划时间、截止时间、责任人、状态"]

  B1 --> C["移动端执行安全检查"]
  C --> C1["选择检查人"]
  C --> C2["逐项选择正常/异常/不适用"]
  C --> C3["异常项填写说明、整改要求、上传照片"]

  C2 --> D{"是否有异常"}
  C3 --> D
  D -- "否" --> E["生成检查记录<br/>任务状态=已完成"]
  D -- "是" --> F["生成检查记录<br/>任务状态=异常待整改"]
  F --> G["生成安全隐患整改工单"]
  G --> H["百数云流程中心办理<br/>整改、验收、关闭"]
  H --> I["检查记录可追溯<br/>任务、提交、明细、隐患"]
  E --> I
```

## 5. 安全检查页面关系

这一张适合解释百数云菜单中的“检查模板、检查任务、检查记录、安全检查”各自做什么。

```mermaid
flowchart LR
  T["检查模板<br/>定义检查类型和检查项"] --> TASK["检查任务<br/>安排谁、何时、检查哪里"]

  FLOW --> REC
```

## 8. 客户演示推荐讲解顺序

```mermaid
flowchart LR
  A["1. 看基础台账<br/>设备、危化品、相关方"] --> B["2. 看点检标准<br/>定义检查什么、怎么判定"]
  B --> C["3. 看点检规则<br/>日检/周检/月检自动生成任务"]
  C --> D["4. 现场扫码报工<br/>正常和异常各演示一次"]
  D --> E["5. 异常进入整改<br/>流程中心闭环"]
  E --> F["6. 安全检查<br/>模板、任务、移动端提交"]
  F --> G["7. 相关方培训<br/>扫码、签字、记录留痕"]
  G --> H["8. 回到记录/底表<br/>展示数据可追溯"]
```

## 9. 演示时可用的话术

- 系统入口在百数云，用户仍然从熟悉的左侧菜单进入，不需要单独记系统地址。
- 设备、危化品、相关方等基础数据是后续任务、报工、培训的源头。
- 点检标准决定“检查什么、怎么判断正常异常”；点检规则决定“什么时候自动生成任务”。
- 现场人员通过扫码进入报工，系统自动匹配当天任务，减少找表和手工录入。
- 一旦发现异常，系统会自动形成隐患整改工单，进入百数云流程中心，实现整改闭环。
- 安全检查模块适合临时或专项检查，移动端执行，异常同样进入整改闭环。
- 相关方培训通过二维码填报和手写签字，形成培训记录和人员签字留痕。
